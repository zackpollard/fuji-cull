Identity — the monospace question

The current UI is monospace throughout — terminal heritage the audience likes. Below: keep it everywhere (current) vs. the proposed hybrid. Both shown as the same top-bar fragment.

WHY — RECOMMEND THE HYBRID

Mono earns its place on data: counters, EXIF, filenames, key hints and log tails are tabular and align beautifully — and it signals "engineered", which is this tool's whole promise. But running prose (guidance, dialogs, errors) reads slower in mono, especially at Dynamic-Type sizes. Keep mono for everything that is a number, a token, or a key; move sentences to IBM Plex Sans. The two are metric siblings, so the seam is invisible. The logo — amber Fuji, cream cap, keep-green bar — is untouched: the tile grammar is the identity.

The tile — core grammar

Every shot is a tile. Fate lives on the bottom edge; live-status overlays sit in the corners so they never fight the decision bar. This must read at 60px, ten across, at arm's length.

anatomy — enlarged
Immich ring · top-right
Teal hollow ring = this shot already exists on the Immich server.
Buffered dot · top-right
Solid blue = full-res copy is cached locally, ready instantly.
Cursor ring · full inset
Amber 2px inset outline + soft glow marks the focused tile.
Decision bar · bottom edge
6px solid green=keep / red=reject. Undecided is a 3px faint hairline — the slot reads as "not yet decided", not empty.
STATE MATRIX
{{ t.label }}
GLANCEABILITY — 56PX, MIXED DECISIONS
WHY — EDGE, NOT CHROME

Decision state is carried by a full-width bar on a fixed edge, so the eye reads a whole row's fates in one sweep without landing on any single tile. Status overlays are tiny and corner-pinned — present when you look, silent when you don't. Nothing here animates on data change (see Motion): a keep is an instant color swap on one bar.

WHY — ENGINEERED, NOT BROKEN

The ~3-minute index happens every connect, so it must feel like work being done: a determinate bar with a live scanning edge, real item counts and throughput, and a running log the user can open. CAMERA SICK is amber, not red — a state to notice, not panic over — and it tells the user what fuji-cull is doing on their behalf.

Motion

Motion lives on the viewer, scrubber and sheets — never on grid cells. With 25k virtualized items, an animation on data change is a jank generator.

Standard easing = cubic-bezier(0.2,0,0,1). prefers-reduced-motion replaces every transition with an instant state change; the scrubber simply appears.

13

Accessibility

CONTRAST

All ink meets AA on --bg; primary text is AAA (16.1:1). Decision colors are verified (keep 10.3:1, reject 6.3:1). Never color-only — keep/reject also differ by bar edge and carry a text label in the viewer.

TOUCH TARGETS

Decision buttons are full-width thumb zones (≥56px) in the touch viewer; controls never below 44×44 (iOS) / 48dp (Android). Tiles stay dense but the whole cell is the tap target.

DYNAMIC TYPE

Prose scales with the system on iOS/Android; mono data scales too but keeps tabular alignment. Tile size is image-driven, independent of type scale, so the grid never reflows under large text — the top bar wraps instead of clipping.

FOCUS & COLORBLIND

Keyboard focus reuses the amber cursor ring — one focus language everywhere. For the green/red pair, the recommended redundant cue is bar-shape (keep solid, reject notched) plus the explicit KEEP/REJECT viewer label.
