WHY — MINIMAL CHROME

No buttons, no menus in the working view — just window bar, counter strip, grid, and a keyboard legend. Every pixel not a photo is overhead. Menus live in the native app menu-bar, out of the way.

WHY — GPU STAGE

The desktop build decodes JPEG on every core and paints through SDL, so the sweep never stalls. The status strip states it plainly — the speed is the product.

WHY — SAME KEYS, NO POINTER

Identical key map to web and iPad-with-keyboard. The stage has no on-screen decision buttons — a desktop culler’s hands never leave the keyboard. The legend footer is the only affordance.

WHY — MPV VIDEO

Video plays through embedded mpv for hardware decode and frame-accurate seek. The same 4 GiB preview wall and “L = pull full clip” contract as every other surface.
